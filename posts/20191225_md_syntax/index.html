Markdown 語法說明

December 25, 2019
Documentation

Markdown 的目標是實現「易讀易寫」。

Markdown 的語法有個主要的目的：用來作為一種網路內容的寫作用語言。也因為這個目標與目的, Markdown 成為許多開源專案的文件的格式, 除了便於可以讓大家不必擔心文件最後呈現的樣子以外，也可以達成document-as-code的版本管控、自動編譯與自動佈署的流水線設計。

標題

讓我們先從不同的標題大小開始。 HTML定義了<h1>—<h6>6種標題大小的階級。<h1>是最高的階級，而<h6>則是最小的階級。

只要在行首插入1到6個#，就能對應到標題1到6階，例如:

# Heading 1
## Heading 2
### Heading 3
#### Heading 4
##### Heading 5
###### Heading 6

Heading 1

Heading 2

Heading 3

Heading 4

Heading 5
Heading 6

段落和換行

一個段落是由一個以上相連接的行句組成，而一個以上的空行則會切分出不同的段落（空行的定義是顯示上看起來像是空行，便會被視為空行。比方說，若某一行只包含空白和 tab，則該行也會被視為空行），一般的段落不需要用空白或斷行縮排。

Paragraph1 is here.
                              <-- 空一行
Paragraph2 is here.

結果展現:

Paragraph1 is here.

Paragraph2 is here.

強調

Markdown 使用星號（*）和底線（_）作為標記強調字詞的符號，被 * 或 _ 包圍的字詞會被轉成用 <em> 標籤包圍，用兩個 * 或 _ 包起來的話，則會被轉成 <strong>，例如：

*single asterisks*

_single underscores_

**double asterisks**

__double underscores__

結果展現:

single asterisks

single underscores

double asterisks

double underscores

連結

Markdown 支援兩種形式的連結語法： 行內(inline)和參考(reference)兩種形式。

不管是哪一種，連結的文字都是用 [方括號] 來標記。

要建立一個行內形式的連結，只要在方塊括號後面馬上接著括號並插入網址連結即可，如果你還想要加上連結的 title 文字，只要在網址後面，用雙引號把 title 文字包起來即可，例如：

This is [an example](http://example.com/ "Title") inline link.

[This link](http://example.net/) has no title attribute.

結果展現:

This is an example inline link.

This link has no title attribute.

如果你是要連結到同樣主機的資源，你可以使用相對路徑：

See my [About](/about/) page for details.

結果展現:

See my About page for details.

圖片

Markdown 使用一種和連結很相似的語法來標記圖片，同樣也允許兩種樣式： 行內和參考。

行內圖片的語法看起來像是：

![Alt text](/path/to/img.jpg)

![Alt text](/path/to/img.jpg "Optional title")

清單

有序清單

有序清單則使用數字接著一個英文句點：

1. First item
2. Second item
3. Third item

結果展現:

  1. First item
  2. Second item
  3. Third item

無序清單

無序清單使用星號、加號或是減號作為清單標記：

* List item
* Another item
* And another item
  • List item
  • Another item
  • And another item

巢狀清單

巢狀清單結合了無序清單與有序清單的標記，並透過tab來作為父子上下關係的區隔子：

* First item
* Second item
    * Second item First subitem
    * Second item second subitem
        * Second item Second subitem First sub-subitem
        * Second item Second subitem Second sub-subitem
        * Second item Second subitem Third sub-subitem
    * Second item Third subitem
        1. Second item Third subitem First sub-subitem
        2. Second item Third subitem Second sub-subitem
        3. Second item Third subitem Third sub-subitem
* Third item

結果展現:

  • First item
  • Second item
    • Second item First subitem
    • Second item second subitem
      • Second item Second subitem First sub-subitem
      • Second item Second subitem Second sub-subitem
      • Second item Second subitem Third sub-subitem
    • Second item Third subitem
      1. Second item Third subitem First sub-subitem
      2. Second item Third subitem Second sub-subitem
      3. Second item Third subitem Third sub-subitem
  • Third item

區塊引言

Markdown使用email形式的區塊引言，如果你很熟悉如何在email信件中引言，你就知道怎麼在Markdown文件中建立一個區塊引言，那會看起來像是你強迫斷行，然後在每行的最前面加上>：

Quoted text. This line is part of the same quote. Also you can put Markdown into a blockquote.

區塊引言(使用inline html)

My goal wasn't to make a ton of money. It was to build good computers. I only started the company when I realized I could be an engineer forever.

— Steve Wozniak

表格

表格並不是Markdown核心規範的一部分，但Hugo支持表格的markdown語法。

| ID  | Make      | Model   | Year |
| --- | --------- | ------- | ---- |
| 1   | Honda     | Accord  | 2009 |
| 2   | Toyota    | Camry   | 2012 |
| 3   | Hyundai   | Elantra | 2010 |

結果展現:

ID Make Model Year
1 Honda Accord 2009
2 Toyota Camry 2012
3 Hyundai Elantra 2010

透過:來對欄位的展示進行向左、靠右或置中的設定。

| ID  | Make      | Model   | Year |
| --- |:---------:|:------- | ----:|
| 1   | Honda     | Accord  | 2009 |
| 2   | Toyota    | Camry   | 2012 |
| 3   | Hyundai   | Elantra | 2010 |

結果展現:

ID Make Model Year
1 Honda Accord 2009
2 Toyota Camry 2012
3 Hyundai Elantra 2010

你也可以使用inline的Markdown標記來強化一些表格值的變化。

| Inline     | Markdown  | In                | Table      |
| ---------- | --------- | ----------------- | ---------- |
| *italics*  | **bold**  | ~~strikethrough~~ | `code`     |

結果展現:

Inline Markdown In Table
italics bold strikethrough code

程式碼區塊

如果要標記一小段行內程式碼，你可以用反引號把它包起來（`），例如：

Use the `printf()` function.

結果展現:

Use the printf() function.

如果程式碼比較長或是有多個段落，那麼可以用(```)來把程式碼給包起來。

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Example HTML5 Document</title>
</head>
<body>
  <p>Test</p>
</body>
</html>

Hugo附帶了一組預定義的短代碼(shortcodes)，這些短代碼是為了方便作者在保持撰寫文件的簡易方便以外並保持Markdown內容乾淨。 以下是利用Hugo的highlight短代碼來對程式碼區塊進行高亮(highlighter)的標示並指定是html的程式碼。

結果展現:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Example HTML5 Document</title>
</head>
<body>
  <p>Test</p>
</body>
</html>

如果想要呈現不同的程式碼的話，請參考List of Chroma Highlighting Languages裡的程式名來進行修改替換。

其它 — abbr, sub, sup, kbd, etc.

除了標準的markdown語法之外，Hugo還支持一些讓文件更加地易於閱讀的語法標註。有興趣的朋友可以適當地使用。

<abbr title="Graphics Interchange Format">GIF</abbr> is a bitmap image format.

H<sub>2</sub>O

C<sub>6</sub>H<sub>12</sub>O<sub>6</sub>

X<sup>n</sup> + Y<sup>n</sup> = Z<sup>n</sup>

Press <kbd>X</kbd> to win. Or press <kbd><kbd>CTRL</kbd>+<kbd>ALT</kbd>+<kbd>F</kbd></kbd> to show FPS counter.

<mark>As a unit of information in information theory, the bit has alternatively been called a shannon</mark>, named after Claude Shannon, the founder of field of information theory.

結果展現:

GIF is a bitmap image format.

H2O

C6H12O6

Xn + Yn = Zn

Press X to win. Or press CTRL+ALT+F to show FPS counter.

As a unit of information in information theory, the bit has alternatively been called a shannon, named after Claude Shannon, the founder of field of information theory.
