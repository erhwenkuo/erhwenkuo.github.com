使用Hugo與Github來建立部落格

December 24, 2019
Documentation

常常會覺得如果有一個地方可以很容易地發佈一些技術文章並且可以自己掌握文章發佈的手法, 那麼就會提高自己寫文章的動機。

在許多公司的技術部落格中發現有一些蠻不錯的工具可以讓我們這種工具人可以快速達到這個心願。今天要介詔的Hugo就歸屬在這個類別。有興趣了解Hugo更詳細用法的朋友們, 請移駕到Hugo官網囉。

以下分為兩部分介紹：建立本地部落格、部署到GitHub。

建立本地部落格

1.先安裝hugo，按照官網的步驟執行安裝(window、Mac都有寫)

2.使用cmd在想要的檔案路徑，建立網址的資料夾之後進入，例如: my_blog

hugo new site my_blog
cd my_blog
git init

3.新增部落格佈景主題，例如:mainroad為例。 以使用Git Submodule的指今來新增連結mainroad專案在themes/mainroad資料夾底下，注意Submodule只是要使用而不是要開發，因為是使用別人做好的主題，我們不需要更改。

git submodule add https://github.com/Vimux/Mainroad.git themes/mainroad

4.建立新文章，例如:檔名為my-first-blog

hugo new posts/my-first-blog.md

這時檔案會在content/posts資料夾裡產生，此檔可使用任意文字編輯器打開markdown文件，將draft改成false，文件內容任意更改。 以下為my-first-blog.md內容：

---
title: "My First Blog"
date: 2019-12-24T15:20:26+08:00
draft: false
categories:
  - "Document"
tags:
  - "Hugo"
  - "Markdown"
---
第一篇測試部落格

5.修改config.toml來指定部落格的baseURL、佈景主題以及相關的插件。 (注意! 以下的設定在不同的佈景主題裡會有不同的配置要求)

baseURL = "https://erhwenkuo.github.io/" #改成你github帳號名稱
languageCode = "zh-tw"
title = "My Hugo Blog"
theme = "mainroad"

[Params]
  subtitle = "二哥 (Erhwen Kuo)" # 部落格的次標題
  description = "緯創IT先進技術實驗室(witlab)" # 部落格的敘述
  opengraph = true
  twitter_cards = false
  readmore = true # 秀出 "Read more" 按鈕在文章列表下
  authorbox = true
  post_navigation = true
  post_meta = ["date", "categories"] # 根據文章的那些meta資訊來決定排序

[Params.sidebar]
  home = "right"   # Configure layout for home page
  list = "right"   # Configure layout for list pages
  single = "right" # Configure layout for single pages
  # Enable widgets in given order
  widgets = ["search", "recent", "categories", "taglist"]

[Params.widgets]
  recent_num = 10 # Set the number of articles in the "Recent articles" widget
  tags_counter = false # Enable counter for each tag in "Tags" widget (disabled by default)

6.在本機運行與測試

hugo server -D

執行後，在瀏覽器輸入網址http://localhost:1313，可看剛建立的部落格網站。

部署到GitHub

1.在GitHub建立兩個repositories

在Github建立my_blog和username.github.io(使用自己的GitHub帳號)

2.建立public資料夾，並連結GitHub 上自己帳號的repositories

在my_blog的目錄裡頭透過git submodule指令來建立一個”public”的資料夾並且把這個目錄連結到github page的repository。原因是Hugo會把相關的markdown檔案進行編譯轉換後的結果就是放置到”public”資料夾。

git submodule add -f https://github.com/erhwenkuo/erhwenkuo.github.com.git public

3.執行Hugo指令來將文件轉譯成HTML格式

執行hugo指令，自動產生編譯檔案新增至public資料夾。

hugo

4.將public資料夾上版到Github上自己帳號的repositories

cd public
git init
git remote add origin https://github.com/erhwenkuo/erhwenkuo.github.com.git
git add .
git commit -m "Blog initial"
git push origin master

5.回到上一層將my_blog資料夾連結到GitHub上的my_blog

cd ..
git init
git remote add origin https://github.com/erhwenkuo/my_blog.git

6.將my_blog資料夾上版

git add .
git commit -m "Hugo blog content initial"
git push origin master

上版後，在瀏覽器輸入自己帳號的網址http://erhwenkuo.github.io，會看到跟本機看到的部落格是一樣的網頁。
